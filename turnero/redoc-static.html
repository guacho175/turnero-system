calendar_buckets_slots_reservar_create

POST /calendar/buckets//slots//reservar

  • valida que event_id pertenezca al bucket (NO confiar en el bot)
  • cambia slot en BD a RESERVADO (sin borrarlo)
  • crea evento confirmado en calendario FINAL del bucket (notificaciones)
Authorizations:
cookieAuthbasicAuthNone
path Parameters
bucket
required
string
event_id
required
string

Responses

/calendar/buckets/{bucket}/slots/{event_id}/reservar

calendar_buckets_sincronizar_create

POST /calendar/buckets/sincronizar

QUÉ HACE: Sincroniza tabla Django (Bucket) con Google Calendar BD.

  • Lee TODOS los eventos de Google BD
  • Extrae buckets únicos presentes
  • Elimina registros de tabla que ya no tienen eventos ("buckets fantasma")
  • Mantiene tabla actualizada después de limpiezas

ENDPOINT: POST /calendar/buckets/sincronizar

BODY: {} // no requiere parámetros

RESPUESTA: { "synced_at": "2026-02-06T...", "buckets_in_google": ["medico", "peluqueria"], "deleted_from_table": ["viejo_bucket"], "deleted_count": 1, "message": "Tabla sincronizada. Se eliminaron 1 bucket(s) sin eventos." }

FLUJO RECOMENDADO:

  1. Limpias eventos con /calendar/calendars/limpiar
  2. Llamas /calendar/buckets/sincronizar
  3. Tabla se actualiza automáticamente
  4. UI carga buckets con /calendar/buckets/tabla (siempre correcto)

PROBLEMA QUE SOLUCIONA: Sin esto: Borras eventos en Google pero tabla Bucket sigue teniéndolos → UI muestra buckets "fantasma" sin eventos Con esto: Tabla siempre refleja realidad de Google

Authorizations:
cookieAuthbasicAuthNone

Responses

/calendar/buckets/sincronizar

calendar_calendars_limpiar_create

POST /calendar/calendars/limpiar

QUÉ HACE: Elimina TODOS los eventos de un calendario Google por ID. Sin especificar rango → borra TODO. Con rango → borra solo ese período.

ENDPOINT: POST /calendar/calendars/limpiar

BODY EJEMPLO: { "calendar_id": "tu_email@gmail.com", "range_start_date": "2026-01-01", // opcional "range_end_date": "2026-12-31" // opcional }

RESPUESTA: { "calendar_id": "...", "deleted_count": 150, "sample_deleted_ids": ["id1", "id2", ...], "range": {"from": "...", "to": "..."} }

CASO DE USO:

  • Limpiar calendario de pruebas rápidamente
  • Después llamar a /calendar/buckets/sincronizar para actualizar tabla
Authorizations:
cookieAuthbasicAuthNone

Responses

/calendar/calendars/limpiar

calendar_calendars_limpiar_bucket_create

POST /calendar/calendars/limpiar-bucket

QUÉ HACE: Elimina SOLO eventos de un BUCKET específico en un calendario. Los eventos se identifican por el field "bucket" en description/properties. Muy útil para limpiar solo un servicio sin tocar otros.

ENDPOINT: POST /calendar/calendars/limpiar-bucket

BODY EJEMPLO: { "calendar_id": "tu_email@gmail.com", "bucket": "medico", // normalizado a minúsculas "range_start_date": "2026-01-01", // opcional "range_end_date": "2026-12-31" // opcional }

RESPUESTA: { "calendar_id": "...", "bucket": "medico", "deleted_count": 50, "sample_deleted_ids": ["id1", "id2", ...], "range": {"from": "...", "to": "..."} }

FLUJO COMPLETO:

  1. Usuario crea slots para "medico" y "peluqueria"
  2. Quiere limpiar solo "medico" (mantiene "peluqueria")
  3. Llama este endpoint con bucket="medico"
  4. Borra solo eventos con bucket=medico
  5. Luego llama /calendar/buckets/sincronizar para actualizar tabla
Authorizations:
cookieAuthbasicAuthNone

Responses

/calendar/calendars/limpiar-bucket
